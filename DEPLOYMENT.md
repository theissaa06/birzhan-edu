# Frame School deployment notes

## Layero fullstack deploy

Layero imports a GitHub repository once and deploys this monorepo as one fullstack project. Keep the project root at `./`; the root `layero.json` defines both units:

- `frontend`: Vite build from `frontend`, published from `frontend/dist` to the CDN.
- `backend`: Node.js runtime from `backend`, listening on port `3003`.
- `api_prefix`: `/api`, proxied by Layero from the public site to the backend container.

The production start command is `npm run start:prod`, which applies Prisma migrations before starting Express. Prisma Client is generated by `postinstall` during dependency installation.

`VITE_API_URL` is optional on Layero: when it is absent in a production build, the frontend uses its own public origin and calls `/api`. Set it only when the API is intentionally hosted on another domain.

For a manual frontend-only upload, upload the contents of `frontend/dist`. A production `index.html` must reference `/assets/index-*.js`; if it references `/src/main.tsx`, Layero is serving Vite source files and the page will stay blank.

Required Layero variables:

- `DATABASE_URL`
- `JWT_SECRET`
- `FRONTEND_URL=https://<layero-public-url>`

Feature-specific variables:

- `TURNSTILE_SECRET_KEY`
- `GEMINI_API_KEY`, `GEMINI_MODEL`
- `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM`
- `R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET`, `R2_PUBLIC_BASE_URL`, `MAX_VIDEO_UPLOAD_MB`
- `PAYMENT_PROVIDER=cloudpayments`, `CLOUDPAYMENTS_PUBLIC_ID`, `CLOUDPAYMENTS_API_SECRET`, `ALLOW_PREMIUM_DEV_OVERRIDE=false`

Optional backend variables:

- `PORT`
- `FRONTEND_URLS` for comma-separated extra allowed origins
- `TRUST_PROXY=1` if Layero does not set `NODE_ENV=production`
- `PUBLIC_BACKEND_URL=https://<backend-public-url>` to let the backend ping its own `/health`
- `KEEP_ALIVE_URLS=https://<backend-public-url>/health` for comma-separated keep-alive/monitoring URLs
- `KEEP_ALIVE_INTERVAL_MS=300000` for a 5-minute ping interval
- `GEMINI_MODEL`
- `ALLOW_TURNSTILE_BYPASS=false`
- `OWNER_ID`, `OWNER_EMAIL` for admin/owner overrides

Password reset emails use a 6-digit code and require working SMTP in production.
Assignment video uploads require Cloudflare R2 env variables and bucket CORS that allows `PUT` from the frontend origin.
Premium activation for CloudPayments-compatible providers is webhook-only; the browser payment callback must not grant access by itself.

## Health and active service

Both `/health` and `/ready` verify PostgreSQL with `SELECT 1`. They return `503` when `DATABASE_URL` is wrong, migrations were not applied, or the database is down; Layero must not mark a backend with a broken database as healthy.

If the Layero plan can sleep inactive services, enable the platform's always-on option or configure an external uptime monitor to call:

```text
https://<backend-public-url>/health
```

The optional `PUBLIC_BACKEND_URL` / `KEEP_ALIVE_URLS` variables add internal pings while the service is awake, but an external monitor is still the reliable way to wake a sleeping free-tier service.

## CORS check

The browser origin must match `FRONTEND_URL` exactly, including protocol and host. `http://localhost:5173` and `http://127.0.0.1:5173` are different origins.

After deploy, open the frontend and check:

- `/` renders the home page.
- `/courses` loads real courses from the backend.
- `/login` and `/register` render without console errors.
- Browser console has no CORS or `Network Error` messages.

## Local smoke test

In one terminal:

```bash
cd backend
npm run db:migrate
npm start
```

In another terminal:

```bash
cd frontend
npm run dev
```

Open `http://localhost:5173/courses`. If courses load, the frontend, backend, Prisma client, and CORS are wired correctly for local development.

If local `/courses` fails with `Can't reach database server at localhost:5432`, start the project-local PostgreSQL cluster or another local PostgreSQL instance that matches `backend/.env`, then run:

```bash
cd backend
npm run db:migrate
node prisma/seed.js
```

Layero frontend variables are embedded during `npm run build`, so changing `VITE_API_URL` requires a new frontend build/deploy.
